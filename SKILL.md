# 多窗口记忆同步系统

## 描述
实时同步多个聊天窗口之间的记忆，解决跨会话记忆隔离问题。每5秒自动更新一次，确保所有窗口都能看到最新的聊天历史和记忆。

## 激活条件
当用户提到以下情况时激活：
- 多窗口聊天记忆不同步
- 跨会话记忆共享
- 实时记忆更新
- 记忆同步系统

## 功能
1. **5秒自动同步**：每5秒检查并更新记忆
2. **跨窗口记忆共享**：在不同聊天会话间同步 MEMORY.md 内容
3. **实时可见性**：随时查看其他窗口的聊天状态
4. **冲突解决**：处理多个窗口同时修改记忆的冲突

## 实现原理
1. 使用文件系统监控（如 inotify）监听 MEMORY.md 变化
2. 通过 OpenClaw 的会话管理 API 获取活跃会话列表
3. 定时（5秒）将记忆更新推送到所有活跃会话
4. 使用版本控制或时间戳解决写入冲突

## 使用方法
```bash
# 启动记忆同步服务
./scripts/start-sync.sh

# 手动触发一次同步
./scripts/sync-now.sh

# 查看同步状态
./scripts/status.sh
```

## 配置文件
`config/sync-config.json`：
```json
{
  "sync_interval_seconds": 5,
  "max_sessions": 10,
  "conflict_resolution": "timestamp",
  "enabled": true
}
```

## 依赖
- OpenClaw 会话管理 API
- 文件系统监控工具
- 定时任务调度

## 注意事项
1. 频繁同步可能增加系统负载
2. 需要处理网络延迟和会话断开情况
3. 敏感信息同步需谨慎
4. 建议设置同步频率根据实际需求调整
